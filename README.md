# Համալսարանական Դեսքթոփ Հավելված - Նախագծի Կառավարման Փաստաթղթեր

## 1. Գործադիր Ամփոփում

### Նախագծի Ընդհանուր Նկարագիր
Համալսարանական դեսքթոփ հավելվածը համապարփակ հարթակ է, որը նախագծված է կրթական փորձը բարելավելու համար՝ տրամադրելով կապի, ռեսուրսների կառավարման և ակադեմիական վարչարարության ինտեգրված գործիքներ։ Հավելվածը ծառայում է որպես կենտրոնացված հարդակ ուսանողների և դասախոսների համար՝ արդյունավետ կերպով կառավարելու իրենց ակադեմիական գործունեությունը։

### Նախագծի Նպատակներ
- Պարզեցնել ուսանողների և դասախոսների միջև հաղորդակցությունը՝ ապահովելով իրական ժամանակի հաղորդագրություններ։
- Հեշտացնել ֆայլերի անվտանգ և համակարգված փոխանակումն ու պահպանումը։
- Կենտրոնացնել դասընթացների կառավարման և առաջադրանքների ներկայացման գործընթացները։
- Բարելավել ժամանակի կառավարումը՝ ինտեգրված օրացույցի և առաջադրանքների կառավարման գործիքների միջոցով։
- Ընդլայնել տեղեկատվության տարածումը միասնական լրատվական հարթակի միջոցով,

### Թիրախային Լսարան
- **Հիմնական օգտատերեր**՝ Համալսարանի ուսանողներ և դասախոսներ
- **Երկրորդական օգտատերեր**՝ Վարչական և դասախոսական անձնակազմ

## 2. Նախագծի Կառավարման Կառուցվածք

### 2.1 Նախագծի Կազմակերպում


**Նախագծի Ղեկավար**՝ Վալերի Միրզոյան
- Նախագծի ընդհանուր համակարգում և իրականացում՝ Վալերի Միրզոյան
- Շահառուների հետ հաղորդակցություն՝  Վալերի Միրզոյան
- Ռիսկերի կառավարում և խնդիրների լուծում՝ Արամ Թևոսյան

**Թիմի Կառուցվածք**՝

1. **Մշակման Թիմ**
   - Գլխավոր ծրագրավորող՝ Վալերի Միրզոյան
   - Frontend ծրագրավորող՝ Դավիթ Հարությունյան
   - Backend ծրագրավորող՝ Վալերի Միրզոյան
   - Տվյալների բազայի ադմինիստրատոր՝ Արամ Թևոսյան
   - DevOps ինժեներ՝ Արամ Թևոսյան

2. **Դիզայնի Թիմ**
   - UI/UX դիզայներ՝ Իդա Ավագյան
   - Գրաֆիկական դիզայներ՝ Իդա Ավագյան

3. **Որակի Ապահովման Թիմ**
   - Որակի ապահովման ղեկավար՝ Արամ Թևոսյան
   - Թեստավորման ինժեներներ՝ Դավիթ Հարությունյան

### 2.2 Դերեր և Պարտականություններ

| Դեր | Պարտականություններ |
|-----|---------------------|
| Նախագծի ղեկավար | Պլանավորում, իրականացում, մոնիտորինգ, շահառուների կառավարում, ռեսուրսների բաշխում |
| Գլխավոր ծրագրավորող | Տեխնիկական ճարտարապետություն, կոդի վերանայում, տեխնոլոգիական որոշումներ, թիմի համակարգում |
| Ծրագրավորողներ | Ֆունկցիոնալության իրականացում, սխալների շտկում, կոդի փաստաթղթավորում, միավորի թեստավորում |
| UI/UX դիզայներ | Ինտերֆեյսի դիզայն, օգտատիրոջ փորձի օպտիմալացում, դիզայնի համակարգի սպասարկում |
| Որակի ապահովման ղեկավար | Թեստավորման ռազմավարության, որակի չափորոշիչներ, թեստային ծրագրի մշակում |

## 3. Նախագծի Կառավարում

### 3.1 Շրջանակի Մեջ Մտնող Ֆունկցիոնալություն

**Փուլ 1 - Հիմնական Ֆունկցիոնալություն** (MVP)
- Օգտատերերի նույնականացում և լիազորում
- Իրական ժամանակի հաղորդակ
- Հիմնական ֆայլերի փոխանակման հնարավորություն
- Նորությունների և հայտարարությունների ցուցադրում
- Դասընթացների ցանկի և ուսումնական պլանների հասանելիություն
- Հիմնական օրացույցի ինտեգրացիա

**Փուլ 2 - Ընդլայնված Ֆունկցիոնալություն**
- հաղորդակի ընդլայնված հնարավորություններ (խմբային հաղորդակ, ֆայլերի փոխանակում հաղորդակում)
- Ֆայլերի տարբերակների վերահսկում
- Առաջադրանքների ներկայացման համակարգ
- Առաջադրանքների ցանկի կառավարում
- Օրացույցի իրադարձությունների հիշեցումներ և համօգտագործում

**Փուլ 3 - Առաջադեմ Ֆունկցիոնալություն**
- Ամպային պահեստի ինտեգրացիա
- Ընդլայնված թույլտվություններ և մուտքի վերահսկում
- Վերլուծություն և հաշվետվություն
- Բջջային հավելված (ապագա դիտարկում)

### 3.2 Շրջանակից Դուրս
- Ֆինանսական գործարքներ մշակում
- Ուսանողների գնահատման համակարգեր (եթե հատուկ չի պահանջվում)
- Գրադարանի կառավարման համակարգեր
- Երրորդ կողմի ուսուցման կառավարման համակարգի (LMS) փոխարինում

### 3.3 Նախագծի իռականացման նախապայմաններ
- Համալսարանը տրամադրում է անհրաժեշտ սերվերային ենթակառուցվածքը կամ ամպային ռեսուրսները
- Active Directory կամ առկա նույնականացման համակարգը հասանելի է ինտեգրման համար
- Համալսարանի օրացույցի համակարգն ունի մատչելի API
- Օգտատերերն ունեն Windows, macOS կամ Linux համակարգիչներ
- Օգտատերերին հասանելի է կայուն ինտերնետ կապ

### 3.4 Սահմանափակումներ
- Նախագիծը պետք է համապատասխանի կրթական տվյալների գաղտնիության կանոնակարգերին (FERPA, GDPR՝ եթե կիրառելի է)
- Սահմանափակ բյուջե՝ [Որոշվելիք է]
- Մշակման ժամկետներ՝ [Որոշվելիք է]
- Նախագիծը պետք է ինտեգրվի համալսարանի առկա համակարգերի հետ

## 4. Նախագծի Ժամանակացույց

### 4.1 Նախագծի Փուլեր

**Փուլ 1՝ Պլանավորում և Վերլուծություն (6-8 շաբաթ)**
- Պահանջների հավաքագրում և վերլուծություն
- Հարցազրույցներ շահառուների հետ
- Տեխնիկական ճարտարապետության նախագծում
- Ռեսուրսների բաշխում
- Ռիսկերի գնահատում

**Փուլ 2՝ Դիզայն (4-6 շաբաթ)**
- UI/UX էսքիզներ և մակետներ
- Տվյալների բազայի սխեմայի նախագծում
- API նախագծում
- Անվտանգության ճարտարապետություն
- Դիզայնի վերանայում և հաստատում

**Փուլ 3՝ Մշակում - Սպրինթային Ցիկլեր (16-20 շաբաթ)**
- Սպրինթ 1-2՝ Նույնականացում և օգտատերերի կառավարում
- Սպրինթ 3-4՝ հաղորդակի ֆունկցիոնալություն
- Սպրինթ 5-6՝ Ֆայլերի փոխանակման համակարգ
- Սպրինթ 7-8՝ Նորությունների, հայտարարությունների և դասընթացների բաժինների ներդնում
- Սպրինթ 9-10՝ Օրացույց և առաջադրանքների հնարավորություններ
- Սպրինթ 11-12՝ Ամպային պահեստի ինտեգրացիա
- Սպրինթ 13-14՝ Ինտեգրացիա և բարելավում

**Փուլ 4՝ Թեստավորում (6-8 շաբաթ)**
- Միավորի թեստավորում (շարունակական)
- Ինտեգրման թեստավորում
- Համակարգի թեստավորում
- Օգտատիրոջ ընդունման թեստավորում (UAT)
- Արդյունավետության և անվտանգության թեստավորում

**Փուլ 5՝ Տեղակայում և Ուսուցում (4-6 շաբաթ)**
- Փորձնական տեղակայում
- Օգտատերերի ուսուցման նստաշրջաններ
- Փաստաթղթավորման ավարտ
- Լրիվ տեղակայում
- Տեղակայումից հետո աջակցություն

**Փուլ 6՝ Սպասարկում և Աջակցություն (Շարունակական)**
- Սխալների շտկում և թարմացումներ
- Ֆունկցիոնալության բարելավումներ
- Օգտատերերի աջակցություն
- Արդյունավետության մոնիտորինգ

### 4.2 Ժամանակացույց

|----------|---------------------|---------|

## 5. Ռեսուրսների Կառավարում

### 5.1 Մարդկային Ռեսուրսներ
- Մշակում՝ 3 անդամ (լրիվ դրույք)
- Դիզայն՝ 1 անդամ (լրիվ դրույք)
- Որակի ապահովում՝ 1 անդամ (լրիվ դրույք)
- Նախագծի կառավարում՝ 1 անդամ (լրիվ դրույք)
- Փաստաթղթավորում և Աջակցություն՝ 3 անդամ (մասնակի դրույքից լրիվ դրույք)

### 5.2 Տեխնիկական Ռեսուրսներ

## 7. Որակի Կառավարում

### 7.1 Որակի Չափորոշիչներ
- Կոդի որակ՝ հետևել սահմանված կոդավորման չափորոշիչներին և լավագույն պրակտիկաներին
- Արդյունավետություն՝ հավելվածի պատասխանի ժամանակը < 2 վայրկյան ստանդարտ գործողությունների համար
- Հասանելիություն՝ 99.5% աշխատաժամանակ ուսումնական կիսամյակներում
- Անվտանգություն՝ անցնել անվտանգության աուդիտը մինչև արտադրական տեղակայումը
- Օգտագործելիություն՝ ձեռք բերել գոհունակության միավոր 4/5 կամ բարձր օգտատերերի թեստավորման մեջ

### 7.2 Որակի Ապահովման Գործունեություններ
- Կոդի վերանայում բոլոր pull request-ների համար
- Ավտոմատացված թեստավորում (միավոր, ինտեգրման)
- Ձեռքով թեստավորում UI/UX-ի համար
- Արդյունավետության թեստավորում
- Անվտանգության թեստավորում և խոցելիությունների սկանավորում
- Օգտատիրոջ ընդունման թեստավորում իրական օգտատերերի հետ

### 7.3 Որակի Ցուցանիշներ
- Կոդի ծածկույթ՝ Նվազագույնը 80%
- Սխալների խտություն՝ < 1 կրիտիկական սխալ 1000 տող կոդի համար

- Համակարգի արդյունավետության ցուցանիշներ

## 8. Հաղորդակցության Կառավարում

### 8.1 Շահառուների Հաղորդակցություն

**Ներքին Թիմի Հաղորդակցություն**՝
- Օրական կարճ հանդիպումներ (15 րոպե)
- Շաբաթական սպրինթի պլանավորում և վերանայում
- Երկշաբաթյա ռետրոսպեկտիվներ
- Slack/Teams շարունակական հաղորդակցության համար

### 8.2 Հաղորդակցության Ուղիներ
- Էլ. փոստ՝ Պաշտոնական հաղորդակցություններ և հաստատումներ
- Նախագծի կառավարման գործիք՝ Առաջադրանքների թարմացումներ և հետևում
- Ակնթարթային հաղորդագրություն՝ Արագ հարցեր և համակարգում
- Վիդեո կոնֆերանս՝ Հեռավար հանդիպումներ և ցուցադրումներ
- Փաստաթղթավորման պահոց՝ Կենտրոնացված նախագծի փաստաթղթեր

### 8.3 Հաշվետվության Պահանջներ
- **Կարգավիճակի հաշվետվություններ**՝ Շաբաթական (ներքին), երկշաբաթյա (շահառուներ)
- **Առաջընթացի հաշվետվություններ**՝ Ամսական մանրամասն առաջընթաց ծրագրի նկատմամբ
- **Խնդիրների հաշվետվություններ**՝ Ըստ անհրաժեշտության, 24 ժամվա ընթացքում կրիտիկական խնդիրների համար
- **Բյուջեի հաշվետվություններ**՝ Ամսական ֆինանսական կարգավիճակ
- **Ռիսկերի հաշվետվություններ**՝ Երկշաբաթյա ռիսկերի ռեգիստրի թարմացումներ

## 9. Փոփոխությունների Կառավարում

### 9.1 Փոփոխությունների Վերահսկման Գործընթաց
1. **Փոփոխության հայտի Ներկայացում**՝ Շահառուն ներկայացնում է պաշտոնական փոփոխության հայտ
2. **Ազդեցության վերլուծություն**՝ Նախագծի թիմը գնահատում է ազդեցությունը շրջանակի, ժամանակացույցի, բյուջեի վրա
3. **Վերանայում**՝ Փոփոխությունների վերահսկման վերանայում է հայտը
4. **Որոշում**՝ Հաստատել, մերժել կամ հետաձգել որոշումը
5. **Իրականացում**՝ Եթե հաստատված է, թարմացնել նախագծի ծրագիրը և հաղորդել
6. **Փաստաթղթավորում**՝ գրանցել փոփոխությունը փոփոխությունների մատյանում

### 9.2 Փոփոխությունների Վերահսկման Խորհուրդ
- Նախագծի Ղեկավար
- Գլխավոր Ծրագրավորող
- Համալսարանի ՏՏ Ներկայացուցիչ
- Նախագծի Հովանավոր կամ պատվիրակ
- Օգտատերերի Ներկայացուցիչ (ուսանող կամ դասախոս)

### 9.3 Փոփոխությունների Շեմեր
- **Մանր փոփոխություններ**՝ Հաստատվում են Նախագծի Ղեկավարի կողմից
- **Միջին փոփոխություններ**՝ Պահանջում են Փոփոխությունների Վերահսկման Խորհրդի հաստատում
- **Մեծ փոփոխություններ**՝ Պահանջում են Ղեկավար Հանձնաժողովի հաստատում

## 10. Տեղակայման և Ուսուցման Ռազմավարություն

### 10.1 Տեղակայման Մոտեցում
- **Փորձնական Փուլ**՝ Տեղակայել փոքր խմբի (500-1000 օգտատեր) համար 2-4 շաբաթով
- **Փուլային Ընդլայնում**՝ Աստիճանաբար ընդլայնել դեպի ստորաբաժանումներ
- **Լրիվ Տեղակայում**՝ Հասանելի դարձնել ամբողջ համալսարանին
- **Վերադարձի Ծրագիր**՝ Պահպանել հետ վերադառնալու հնարավորությունը կրիտիկական խնդիրների դեպքում

### 10.2 Ուսուցման Ծրագիր
- **Օգտատերերի Փաստաթղթավորում**՝ Համապարփակ օգտատերերի ուղեցույցներ և ՀՏՀ
- **Վիդեո Դասընթացներ**՝ Կարճ վիդեոներ յուրաքանչյուր հիմնական ֆունկցիայի համար
- **Ուղիղ Ուսուցման Նստաշրջաններ**՝ Նախատեսված սեմինարներ ուսանողների և դասախոսների համար
- **Օգնության Սեղան**՝ Նվիրված աջակցության թիմ նախնական տեղակայման ընթացքում
- **Ուսուցիչների Ուսուցում**՝ Պատրաստել ստորաբաժանումների ներկայացուցիչներին օգտատերերին օգնելու համար

### 10.3 Աջակցության Կառուցվածք
- **Մակարդակ 1**՝ Օգնության սեղան հիմնական հարցերի և խնդիրների համար
- **Մակարդակ 2**՝ Տեխնիկական աջակցություն բարդ խնդիրների համար
- **Մակարդակ 3**՝ Մշակման թիմ սխալների շտկման և համակարգային խնդիրների համար

### 11.1 Նախագծի Հաջողության Ցուցանիշներ
- Հավելվածը մատակարարված է ժամանակին և բյուջեի սահմաններում
- Բոլոր հիմնական ֆունկցիաները գործող են և բավարարում են պահանջները
- Հաջող ավարտված UAT շահառուների ստորագրությամբ
- Անվտանգության և արդյունավետության չափորոշիչները բավարարված են
- Օգտատերերի ընդունման մակարդակը 70% առաջին կիսամյակի ընթացքում

### 11.2 Տեղակայումից Հետո Հաջողության Ցուցիչներ
- Օգտատիրոջ գոհունակության միավորը ≥ 4/5
- Համակարգի աշխատաժամանակը ≥ 99.5%
- Միջին պատասխանի ժամանակը < 2 վայրկյան
- Աջակցության տոմսերի լուծումը SLA-ի սահմաններում
- Ակտիվ ամսական օգտատերերը ≥ 80% թիրախային լսարանից

## 12. Նախագծի Կառավարում

### 12.1 Որոշումների Կայացման Լիազորություն
- **Ռազմավարական Որոշումներ**՝ Նախագծի Ղեկավար
- **Նախագծային Մակարդակի Որոշումներ**՝ Նախագծի Ղեկավար հովանավորի հաստատմամբ
- **Տեխնիկական Որոշումներ**՝ Գլխավոր Ծրագրավորող թիմի ներդրումով
- **Գործառնական Որոշումներ**՝ Նախագծի Ղեկավար

### 12.2 Հանդիպումների Կառուցվածք
- **Օրական Կարճ Հանդիպումներ**՝ Մշակման թիմ (15 րոպե)
- **Սպրինթի Պլանավորում**՝ Յուրաքանչյուր 2 շաբաթը մեկ (2-4 ժամ)
- **Սպրինթի Վերանայում**՝ Յուրաքանչյուր սպրինթի վերջում (1-2 ժամ)
- **Ղեկավար Հանձնաժողով**՝ Ամսական (1-2 ժամ)
- **Կարգավիճակի Հանդիպումներ**՝ Շաբաթական հիմնական շահառուների հետ (30-60 րոպե)

## 13. Հավելվածներ

 Տերմինների Բառարան
- **MVP**՝ Նվազագույն Կենսունակ Արտադրանք
- **UAT**՝ Օգտատիրոջ Ընդունման Թեստավորում
